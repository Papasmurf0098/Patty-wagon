# Patty-wagon

Patty-wagon now starts from a clean, organized baseline instead of an empty repository.

## What changed

This overhaul introduces:

- a clear project overview and onboarding guide;
- a documented repository layout;
- contribution conventions for future work;
- basic repository hygiene files to reduce accidental clutter.

## Repository layout

```text
.
├── README.md
├── CONTRIBUTING.md
├── Makefile
├── .editorconfig
├── .gitignore
└── docs/
    └── ARCHITECTURE.md
```

## Quick start

1. Read `README.md` for the repository purpose.
2. Review `docs/ARCHITECTURE.md` before adding code or services.
3. Follow `CONTRIBUTING.md` for conventions.
4. Use `make help` to see the current developer workflow placeholders.

## Current status

This repository is intentionally scaffolded but code-light. The next step is to add the application or service that Patty-wagon is meant to contain.
