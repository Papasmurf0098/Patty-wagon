# Contributing

## Goals

Keep the repository easy to navigate, easy to run, and easy to review.

## Guidelines

- Prefer small, focused changes.
- Document new top-level directories in `README.md`.
- Add or update architecture notes in `docs/ARCHITECTURE.md` when introducing major components.
- Avoid committing generated files, secrets, or local environment artifacts.
- Include concise setup and validation steps in pull requests.

## Recommended workflow

1. Create a focused branch.
2. Make changes in the smallest useful increment.
3. Run relevant checks.
4. Update documentation when behavior or structure changes.
5. Open a pull request with a short summary and validation notes.
